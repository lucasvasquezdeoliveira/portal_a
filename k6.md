# Estratégia de Massa de Dados com K6

Para garantir que os testes de carga possam ser executados com eficiência, optamos por utilizar dados estáticos e reais durante os testes para os endpoints.

1. **Massa de Dados**
   - Para o endpoint `/flip_coin.php`, não é necessário uma massa de dados complexa, já que a operação é basicamente aleatória.
   - Para o endpoint `/my_messages.php`, os testes utilizam um conjunto de dados pré-existentes no sistema para simular consultas e validação das mensagens dos usuários.

2. **Limpeza de Dados**
   - Como os dados manipulados são de leitura e aleatórios, não haverá necessidade de limpeza massiva após os testes.
   - Os dados de mensagens poderiam ser validados contra valores reais e predefinidos para garantir consistência nos testes de retorno da API.
